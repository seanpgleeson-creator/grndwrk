# impeccable — Design Context

## Design Context

### Users
Single-candidate users using grndwrk to run a deliberate, proactive job search campaign.
Primary context: they are making important career decisions and need clarity, confidence, and consistency across research, scoring, and application materials.

### Brand Personality
Premium, focused, editorial, and a little serious.
Emotional targets: calm confidence, trustworthiness, and "deliberate" pacing — closer to Linear or Notion than a generic SaaS dashboard. No noisy, flashy, or feature-heavy feel.

### Aesthetic Direction
Linear-inspired UI: fixed left sidebar (~220px), full-width main content with generous whitespace, no centered modal cards as primary content areas.
**Light mode is the default**; dark mode is opt-in via sidebar footer toggle.

Typography (canonical for future work):
- Body/UI font: DM Sans
- Display/headings: Fraunces
- Hierarchy: page titles large and confident (Fraunces 28px); section labels small, uppercased, semibold; labels above fields; comfortable body text.

Color:
- Light default tokens: background `#FAFAF8`, sidebar `#F4F4F2`, surface `#FFFFFF`, text `#1A1A1A`, muted `#6B6B6B`, border `#E5E5E5`
- Dark tokens: background `#0F0F0F`, sidebar `#161616`, surface `#1A1A1A`, text `#E5E5E5`, muted `#6B6B6B`, border `#2A2A2A`
- Accent: `#3B4F7C` (slate blue); used only for active states, CTAs, and focus rings. No gradients on primary backgrounds.

Layout rules:
- Full-width layout. No centered modal cards on dark/light canvases for primary content.
- Left sidebar navigation (fixed 220px), main content fills the rest.
- Lowercase `grndwrk` wordmark at top of sidebar in Fraunces.
- No top navigation bar.
- Onboarding/setup flows feel like a guided experience (sidebar step list) rather than a form inside a box.
- Cards/panels use borders (no drop shadows); consistent rounded corners (cards 8px, inputs 6px, badges 4px).

### Design Principles
1. Confident, not flashy: predictable UI patterns and minimal decorative chrome.
2. Focused, not feature-heavy looking: fewer, larger content regions; generous whitespace but not empty.
3. Trustworthy for serious decisions: typography hierarchy and restraint signal credibility.
4. Consistency across the system: semantic tokens and reusable component styling; avoid hardcoded hex colors in UI components.
5. Calm interaction feedback: 150ms ease transitions for hover and mode toggle; respect reduced motion preferences; keep focus states clearly visible (accent color, no heavy shadows).
